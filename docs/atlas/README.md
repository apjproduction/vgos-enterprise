# Project Atlas

Project Atlas is the APJ Platform specification and operating doctrine. It defines how APJ Labs thinks about products, intelligence, architecture, engineering, user experience, connectors, agents, and long-term platform governance.

This documentation should be treated as a living source of truth for VGOS, VidMaker, TradeWizAI, DriveMart, and future APJ Labs products.

## Purpose

APJ Labs is not only building isolated SaaS products. APJ Labs is building reusable intelligence infrastructure that helps founders and organizations observe, reason, decide, execute, measure, learn, remember, and improve.

Project Atlas exists so that every human contributor, AI coding agent, product strategist, designer, and future engineer builds from the same vision, vocabulary, architecture, and standards.

## Core Thesis

> APJ Labs builds intelligence infrastructure that helps founders and organizations make better decisions through reusable executive intelligence.

VGOS is the first major expression of this platform philosophy. VidMaker, TradeWizAI, DriveMart, and future APJ Labs products should increasingly consume the same intelligence kernel, connector framework, agent runtime, identity model, and executive experience patterns.

## Documentation Rule

No major architecture decision, product capability, connector, agent, or strategic feature should be implemented without updating the relevant Project Atlas volume, ADR, capability manifest, or RFC.
